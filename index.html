Topology

Objective

This lab will be split to 3 diferent problems which you will need to solve in the assigned order

  • The customer has added to his existing GWLB environment an App Server in a new VPC
  • There are several issues faced since this addition

Lab 4 - Problem A

Lab objectives

  • After connecting the App VPC, the App server can't be reached
  • Goal:
    • Find the root cause of the issue
    • Manage to connect with HTTP from your computer to the app server

Lab 4 - Problem B

Lab objectives

  • The App server can't connect to the Web server
  • Goal:
    • Resolve communicating between the App server and the Web server
    • Manage to ping between the 2 servers

Lab 4 - Problem C

Lab objectives

  • Traffic between the App server and Web server are not hitting the correct rule
  • There is a specific rule in the policy made for App-Web traffic. but traffic is hitting the any-any accept rule
  • Goal:
    • Find the reason for the wrong rule being hit
    • Make the relevant adjustments so that the traffic hits the correct rule


Post Deployment Actions

Step 1

Make sure the MGMT server has finished FTW.

  • Connect with SSH to the MGMT server
  • If you are in expert mode, the machine is done with FTW. If not, wait until the VM finishes FTW and automatically reboots again.
  • Check that the MGMT is completely up by running api status | Do not proceed forward before this is ready!

Step 2

Run the Auto-MGMT script.

  • Connect with SSH to the MGMT server
  • Run the following command on the MGMT server: /var/log/automgmt.sh
